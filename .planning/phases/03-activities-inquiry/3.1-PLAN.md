## Pattern from Existing Collections

The Activities collection should follow the same patterns as Leads and PropertiesStub:
- Use `CollectionConfig` from `payload`
- Admin configuration with `useAsTitle` and `defaultColumns`
- Access control: public read, admin create/update/delete
- Fields organized with tabs for admin UX
- Relationship field linking to properties

## Activities Collection Context (from ROADMAP.md Phase 3)
- Fields: title, description, price, media (images), availability, linkedProperties (relationship)
- Access: public read, admin create/update/delete
- Availability can be JSON initially (calendar UI in Phase 5)

  <action>
    Create a new file payload/src/collections/Activities.ts with a Payload CMS collection for Airbnb experiences.

    The collection MUST include:

    1. **Admin configuration:**
       - `useAsTitle: 'title'`
       - `defaultColumns: ['title', 'price', 'status', 'createdAt']`
       - `listSearchableFields: ['title', 'description']`

    2. **Access control:**
       - `read: () => true` (public read for published activities)
       - `create: ({ req: { user } }) => Boolean(user)` (admin only)
       - `update: ({ req: { user } }) => Boolean(user)` (admin only)
       - `delete: ({ req: { user } }) => Boolean(user)` (admin only)

    3. **Fields organized in TABS:**

    **Tab: "Basic Info"**
    - title (text, required) - admin description: 'Activity or experience name'
    - shortDescription (text) - admin description: 'Brief tagline for listings (max 150 chars)'

    **Tab: "Details"**
    - description (textarea) - admin description: 'Full description of the experience'
    - highlights (array) - admin description: 'Key highlights of this experience':
      - highlight (text, required)
    - duration (text) - admin description: 'Duration (e.g., "2 hours", "Full day")'
    - groupSize (group):
      - minGuests (number, min: 1)
      - maxGuests (number, min: 1)

    **Tab: "Pricing"**
    - price (number, required) - admin description: 'Price per person (INR)'
    - currency (select, defaultValue: 'INR'): INR, USD, EUR
    - includes (textarea) - admin description: 'What is included in the price'
    - cancellationPolicy (textarea) - admin description: 'Cancellation and refund policy'

    **Tab: "Media"**
    - gallery (relationship, relationTo: 'media', hasMany: true, isSortable: true) - admin description: 'Activity photos (drag to reorder)'
    - featuredImage (relationship, relationTo: 'media') - admin description: 'Main image shown in listings'

    **Tab: "Availability"**
    - availability (json) - admin description: 'Availability schedule (Phase 5 will add calendar UI)'
    - bookingLeadTime (number) - admin description: 'Minimum advance booking time in hours'

    4. **Outside tabs (sidebar):**
    - status (select, defaultValue: 'draft', required, position: 'sidebar'): Draft, Published, Archived
    - linkedProperties (relationship, relationTo: 'properties', hasMany: true) - admin description: 'Properties this activity is associated with'

    5. **Indexes:**
    - { status: 1 }
    - { price: 1 }

    Export the collection as `Activities`.
