<task type="execute">
  <name>Task 1: Create docker-compose.yml with Postgres and App services</name>
  <files>docker-compose.yml</files>
  <action>
    Create docker-compose.yml at project root with the following services:

    1. **postgres service:**
       - image: postgis/postgis:15-3.4
       - container_name: real-estate-postgres
       - environment:
         - POSTGRES_USER: postgres
         - POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-postgres}
         - POSTGRES_DB: real-estate-saas
       - ports: ["5432:5432"]
       - volumes:
         - postgres_data:/var/lib/postgresql/data
       - healthcheck:
           test: ["CMD-SHELL", "pg_isready -U postgres"]
           interval: 5s
           timeout: 5s
           retries: 5
           start_period: 10s

    2. **app service:**
       - build context: . (Dockerfile in root)
       - container_name: real-estate-app
       - depends_on: postgres with condition service_healthy
       - environment:
         - DATABASE_URL: ${DATABASE_URL}
         - PAYLOAD_SECRET: ${PAYLOAD_SECRET}
         - PAYLOAD_PUBLIC_SERVER_URL: ${PAYLOAD_PUBLIC_SERVER_URL}
         - NODE_ENV: development
       - ports: ["3000:3000"]
       - volumes:
         - .:/app
         - /app/node_modules
         - /app/.next
       - healthcheck:
           test: ["CMD-SHELL", "wget --no-verbose --tries=1 --spider http://localhost:3000 || exit 1"]
           interval: 30s
           timeout: 10s
           retries: 3
           start_period: 60s

    3. **Volume declaration at top level:**
       - postgres_data (no external, driver local)

    4. **Named volumes syntax** for node_modules and .next to avoid overwriting in bind mount

    Note: Dockerfile will be created in Plan 1.2 (Payload Integration). This docker-compose references it but does not create it.
  </action>
  <verify>
    <automated>grep -c "postgis/postgis:15-3.4" docker-compose.yml && grep -c "service_healthy" docker-compose.yml && grep -c "postgres_data" docker-compose.yml</automated>
  </verify>
  <done>
    docker-compose.yml exists with postgres (postgis/postgis:15-3.4) and app services, healthchecks on both, volumes for persistence, app depends_on postgres with service_healthy condition
  </done>
</task>
